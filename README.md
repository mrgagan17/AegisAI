# AegisAI -AI Customer Support Agent Platform (RAG-Powered)

A modern, containerized AI Customer Support Agent and Copilot Platform. The system implements a complete **RAG (Retrieval-Augmented Generation)** pipeline. Administrators can upload company reference manuals (e.g., employee handbooks, FAQs), which are chunked, embedded, and stored in a vector database. Standard users can then ask questions in real-time, receiving answers strictly grounded in the uploaded documents using Google's Gemini models.

---

## 🚀 Key Features

* **Real-time AI Copilot**: Streaming chat interface powered by WebSockets to query reference documents.
* **Document Processing & Chunking**: Automatic parsing of PDF and text files, chunking, and embedding.
* **Semantic Vector Search**: Powered by a self-hosted ChromaDB instance to find the most relevant chunks.
* **Grounded AI Answers**: Utilizes the Google Gemini API (`gemini-3.5-flash` and `gemini-embedding-2`) to generate answers based on context retrieved from the vector store.
* **Onboarding & Authentication**: Secured using JSON Web Tokens (JWT) with separate dashboard views for Admins (upload, delete, manage users) and Users (AI Copilot chat).
* **Responsive Styling**: Glassmorphic theme support (Light/Dark mode) styled using Tailwind CSS and Lucide React.

---

## 🛠 Tech Stack

* **Frontend**: React (Vite, Tailwind CSS, Lucide icons, React Router, React Hook Form)
* **Backend**: Node.js, Express, WebSocket (`ws`)
* **Vector Store**: ChromaDB (Official Docker container)
* **Primary Database**: MongoDB (Official Docker container)
* **AI Engine**: Google Gemini API

## 🔧 Installation & Local Setup

### 1. Clone & Set Environment Variables
Create a `.env` file in the **root** folder and add the following:

```env
PORT=5000
MONGODB_URI=mongodb://localhost:27017/ai-customer-support
JWT_SECRET=your_jwt_access_secret_key
JWT_REFRESH_SECRET=your_jwt_refresh_secret_key
GEMINI_API_KEY=your_google_gemini_api_key_here
NODE_ENV=development

# Vector store (ChromaDB)
VECTOR_DB=chroma
CHROMA_HOST=localhost
CHROMA_PORT=8000
CHROMA_COLLECTION=documents
```



### 2. Run Databases using Docker Compose
Start MongoDB and ChromaDB by running the following command from the root directory:

```bash
docker compose up -d
```
This will start:
* **MongoDB** on `localhost:27017`
* **ChromaDB** on `localhost:8000`
* **Backend Node API** container on `localhost:5000`

### 3. Run Backend (Development Mode)
If you want to run the backend node process locally outside Docker (e.g. for debugging):

```bash
cd backend
npm install
npm run dev
```

### 4. Run Frontend (Development Mode)
Open a new terminal window, navigate to the frontend folder, and launch the Vite dev server:

```bash
cd frontend
npm install
npm run dev
```

The frontend will start running at **`http://localhost:5173/`**.

## 🧪 Verification & Diagnostics

### Run RAG Diagnostics
You can verify that your backend, ChromaDB, and the Gemini API are talking to each other correctly by running:

```bash
cd backend
node scripts/test-rag.js
```
This test splits text, generates embeddings, saves them to Chroma, queries them, and prints a sample response generated by Gemini.
